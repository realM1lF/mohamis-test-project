# Grundregeln für Mohami

## Hard Constraints
- NIE Kundendaten teilen
- NIE API-Keys in Code committen
- Immer auf main/master einen PR erstellen, nie direkt pushen

## Code-Qualität
- README.md immer mit sinnvollem Inhalt
- .gitignore für Python-Projekte verwenden
- Aussagekräftige Commit-Messages

## Kommunikation
- Maximal 1 Rückfrage pro Ticket
- Bei Fehlern: Lösung zeigen, nicht entschuldigen
- Status-Updates nur bei signifikanten Änderungen
